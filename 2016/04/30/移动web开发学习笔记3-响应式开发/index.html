移动web开发学习笔记3-响应式开发

发表于 2016-04-30   |   分类于 移动web   |     |  阅读量 次

之前总结了移动web开发的一些基础知识点。下面总结一下在进行移动web开发的时候需要注意的一些问题。

响应式设计

flex布局

flex布局是css3的一个重要特性。关于flex布局的具体内容，可以查看阮一峰大神的博客,链接

媒体查询

媒体查询是响应式设计的一个重点。什么是媒体查询呢？媒体查询就是在css样式中，首先判断不同的设备，为不同的设备设置不同的css属性。这里对设备的判断不需要用js来完成，而是直接由css完成的。媒体查询的语法如下

@media screen and (min-width:640px) and (max-width:1024px){
    body{
        background-color:red
    }
}

上面的例子表示的意思是：当屏幕的宽度大于等于640px，小于等于1024px的时候，body的背景颜色为红色。媒体查询也可以针对不同的媒体使用不同的stylesheets:

<link rel="stylesheet" media="screen and (min-width:800px) and (max-width:1024px)" href="mystyle.css">

在媒体查询中，media 后面紧跟的表示媒体类型，其值可以为：screen(屏幕)，print（打印机），handheld（手持设备），all（通用） 等等。一般为 screen。后边括号里的 min-width 表示媒体功能，其他的值参考这里。两者中间的and被称为关键字，其他关键字还包括not ( 排除某种设备 ) ,only ( 限定某种设备 ) .

注意事项

  • 使用百分比布局
  • 使用弹性图片。所谓弹性图片，即给图片加上CSS样式：max-width:100%. 然后再其外部加上一个div，将图片包裹。该div需要设置响应式样式。这样的话，图片就可以随着页面大小的变化而等比缩放。需要注意的是，如果给图片设置width:100%，则比div宽度小的图片也会被等比放大成外部div的宽度。
  • 重新布局，注意显示和隐藏不必要的元素。经常需要切换位置的元素可以将其设置为绝对定位，这样可以提高渲染的性能。

特殊样式处理

高清图片的处理

在移动web页面上渲染图片，为了避免图片产生模糊，图片的宽高应该按照物理像素的单位进行渲染，图片才出现模糊。所以，我们在设置图片的css宽高的时候，需要根据当前手机屏幕的dpr来处理。即：

width:(w_value/dpr) px
height:(h_value/dpr) px

一像素边框

在高清屏幕上，想要渲染出一像素的边框，如果使用border-width:1px来渲染，是回出现问题的。因为在高清屏幕上，手机会用两个dp来渲染一个px。因此，渲染出来的border看起来就比”一像素”粗。如何解决这个问题呢？首先想到的是border-width:0.5px。但是这种写法只有iOS8支持。解决方案是：

使用伪类，加上scaleY(0.5)来进行缩放。
例子如下：

    div{
        position:relative;
    }
    div:before{
        position:absolute;
        top:-1px;
        left:0;
        content:"";
        width:100%;
        height:1px;
        border-top:1px solid black;
        -webkit-transform:scaleY(0.5);
    }

使用rem来处理元素的大小

rem:根据根节点的font-size为相对单位。

em:根据父亲节点的font-size为相对单位。在多层嵌套下，变得难以维护。

rem的基值设置为多少比较好？

rem = screen.width/20（仅供参考，具体情况，具体分析）

注意，一般来讲，font-size是不应该使用rem等相对单位的。因为字体的大小是趋于阅读的实用性，并不适用于排版布局。同理，趋向于一些固定的元素的特性，不实用rem而改为使用px去确保在不同屏幕上表现一致。

多行文本溢出

单行文本溢出…

line{
    overfolw:hidden;
    white-space:nowrap;
    text-overflow:ellipsis;
}

多行文本溢出…

line{
    display:-webkit-box;
    overflow:hidden;    
    text-overflow:ellipsis;
    word-break:break-all;
    -webkit-box-orient:vertical;
    -webkit-line-clamp:2 //这里的属性值表示显示的行数
}
